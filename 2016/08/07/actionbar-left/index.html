关于使用Actionbar时右侧功能部分的定义

项目中很多activity都会用到相同或类似的顶部标题栏，我们可以写好一个固定的、完整度比较高的布局，然后在每个界面的布局文件中进行include。但V7包中的actionbar能更好解决这个方案，让你的activity继承AppCompatActivity即可。

下面的方法用来定义顶部标题栏的background 

if (GlobalApplication.getInstance().getResources().getBoolean(R.bool.openLeader)){
 getSupportActionBar().setBackgroundDrawable(getResources().getDrawable(R.drawable.ac_bg));}

在activity中重写下面的三个方法，即可定义标题栏右侧的功能区，功能区一般都是增删改查的功能。

此方法是用来定义标题栏右侧的布局的，需要在res目录下写好一个布局。 

public boolean onCreateOptionsMenu(Menu menu) {
    getMenuInflater().inflate(R.menu.menu_enforce, menu);
    return super.onCreateOptionsMenu(menu);
}

因为好多的界面都会使用项目的顶部布局，但不是所有界面的顶部功能都是一致的。因此下面的方法就是用来隐藏一些不需要用到的功能的。

public boolean onPrepareOptionsMenu(Menu menu) {
    menu.findItem(R.id.menu_id_submit).setVisible(true);
    return super.onPrepareOptionsMenu(menu);
}

点击标题栏的功能按钮后，所执行的操作，就是在下方法中实现的。 

public boolean onOptionsItemSelected(MenuItem item) {
    int itemId = item.getItemId();
    if (itemId == R.id.menu_id_submit) {
   //提交
   }else if(itemId == R.id.menu_id_edit){
   //修改
   }
   return true;
}

Menu的布局文件就比较简单了，这里只做简单介绍，具体的功能参照实际需求进行修改。 

<?xml version="1.0" encoding="utf-8"?>
<menu xmlns:android="http://schemas.android.com/apk/res/android"
xmlns:ecommunity="http://schemas.android.com/apk/res-auto" >

<item
    android:id="@+id/menu_id_submit"
    android:orderInCategory="1"
    android:title="@string/submit"
    android:visible="true"
    ecommunity:showAsAction="always"/>
<item
    android:id="@+id/menu_id_edit"
    android:orderInCategory="1"
    android:title="@string/submit"
    android:visible="true"
    ecommunity:showAsAction="always"/>

</menu>
